# Business Logic Reference

โฟลเดอร์นี้เก็บเอกสารอธิบาย "กฎ/สูตร/เงื่อนไข" ของ business logic แต่ละโดเมนใน xInvest **ตามสภาพปัจจุบัน** ของโค้ด (ไม่ใช่ประวัติการเปลี่ยนแปลง — ประวัติ/เหตุผลของการตัดสินใจอยู่ใน [docs/adr/](../adr/))

เป้าหมาย: ให้นักพัฒนา/ทีมเทคนิคภายนอกอ่านแล้วเข้าใจว่าระบบคำนวณ/ตัดสินใจอย่างไร โดยไม่ต้องไล่โค้ดทั้งหมดเอง

## เมื่อไหร่ถึงจะมีเอกสารในนี้

เอกสารในโฟลเดอร์นี้ถูกสร้าง/อัปเดตเฉพาะเมื่อมีการสั่งงานที่ระบุชัดเจนว่าให้ "ทำเอกสารด้วย" ไม่ใช่ทุก task ที่แตะ business logic

## รูปแบบไฟล์

หนึ่งไฟล์ต่อหนึ่งโดเมน ตั้งชื่อแบบ camelCase ให้สอดคล้องกับ `docs/features/` ที่มีอยู่ เช่น:

- `fundAnalysis.md` — สูตรคำนวณ Sharpe/Treynor/Sortino, เกณฑ์ AI insight
- `workflowApproval.md` — flow การอนุมัติคำขอ, สถานะ, ผู้มีสิทธิ์อนุมัติ
- `investorCalculation.md` — การคำนวณยอดพอร์ต, transaction, statement

แต่ละไฟล์ควรมีโครงหัวข้อประมาณนี้:

```markdown
# <ชื่อโดเมน>

## สรุป
อธิบายสั้น ๆ ว่าโดเมนนี้ทำหน้าที่อะไร

## กฎ/สูตร
รายละเอียดกฎทางธุรกิจ สูตรคำนวณ เงื่อนไข edge case

## Reference โค้ด
ลิงก์ไฟล์:บรรทัด ที่ implement กฎแต่ละข้อ เช่น
`backend/fundDecision/tasks.py:42` — คำนวณ fund impact summary

## ที่เกี่ยวข้อง
ลิงก์ไปยัง ADR ที่เกี่ยวข้อง (ถ้ามี) ใน docs/adr/
```

เอกสารนี้ถูก **เขียนทับ** เมื่อกฎเปลี่ยน ไม่สะสมเวอร์ชันเก่า — ถ้าต้องการรู้ว่าทำไมถึงเปลี่ยน ให้ดู ADR
