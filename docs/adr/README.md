# Architecture / Business Decision Records (ADR)

โฟลเดอร์นี้เก็บบันทึกการตัดสินใจสำคัญที่เกี่ยวกับ business logic หรือสถาปัตยกรรมของ xInvest แต่ละไฟล์คือ**เหตุการณ์เดียว ณ เวลาหนึ่ง** ไม่แก้ไขย้อนหลัง — ถ้าการตัดสินใจเปลี่ยนในภายหลัง ให้เขียน ADR ใหม่แล้วอ้างอิงกลับไปยังไฟล์เดิม

ต่างจาก [docs/business-logic/](../business-logic/) ตรงที่ business-logic docs อธิบาย "กฎเป็นอย่างไรตอนนี้" ส่วน ADR อธิบาย "ทำไมถึงตัดสินใจแบบนี้ ตอนไหน โดยใคร"

## เมื่อไหร่ถึงจะมี ADR

สร้างเฉพาะเมื่อสั่งงานที่ระบุชัดเจนว่าให้ "ทำเอกสารด้วย" **และ** การเปลี่ยนแปลงนั้นมีนัยสำคัญ (เปลี่ยนสูตรคำนวณ, เปลี่ยน flow อนุมัติ, เปลี่ยนวิธีจัดการข้อมูลการเงิน ฯลฯ) — การแก้ไขเล็กน้อย (fix typo, refactor ที่ไม่เปลี่ยนพฤติกรรม) ไม่ต้องมี ADR
